### Wayfair 广告

- 入口文档：`docs/finebi/FineBI广告数据梳理.md`、`docs/business/指标口径字典.md`、`docs/playbooks/广告组月报生成流程.md`。

- 2026-07-07 补充：BID 覆盖率数据库侧已复核。2026-01 至 2026-05 广告事实按 `campaignlayer_id = campaign_id` 匹配出价快照 `124,503` 行，高于按名称匹配的 `121,437` 行；`WF-TD-CA`、`WF-TD-UK` 在 `wf广告出价表` 中未见店铺快照，不能未经确认用 `WF-TD` BID 回填。
- 2026-07-07 状态过滤补充：广告事实表中 `isB2b=true` 和非 `ACTIVE/ACTIVE` 状态组合均有非零花费和 WSC；默认智能取数不要机械过滤 `isB2b`、`campaign_status`、`product_status`，除非用户明确指定或复刻具体 FineBI 组件。

### Overstock 库存 / 价格

- 当前产品和基础成本查 `os_manage_cost_latest`；按日 / 未来促销上传价查 `osprice`；`link_sku_price_scj` 每天 09:30、14:30 消费上传价。活动重叠按有效价格最小值，非促销基础成本只兜底当天。

### Wayfair 库存 / 有货率 / 断货

- 入口文档：`docs/finebi/FineBI库存数据梳理.md`、`docs/playbooks/有货率报表生成流程.md`。
- 核心对象：`warehouse_stock_在售`、`warehouse_stock_最新`、`库存拆分记录`、`平台产品详情表`、`wayfair上架sku销售映射`、`仓库映射表`、`tb_kyxl_forecast_and_reality`、`考虑放弃sku清单`。
- 选表规则：历史库存优先 `warehouse_stock_在售`；该表只硬过滤四类一级分类，不自动删除停用 SKU，且每日 04:05 后的源记录存在日内时差。当前库存优先每小时 `:05` 按每仓最大日期重建的 `warehouse_stock_最新`；`库存拆分记录` 当前停在 2026-07-08，恢复前不用于当前拆分库存；预测断货分日查 `tb_kyxl_forecast_and_reality`，月度 KPI 查物理表 `断货sku数量` / FineBI `zzdnb_断货sku数量`。
- 有货率规则：Wayfair 采购 SKU 映射优先 `wayfair上架sku销售映射`，缺失时回退 `平台产品详情表.采购sku`；组合 SKU 任一组件不可用即断货。
- 库存运营链补充：`库存最终看板`、平台库存数量推送、鲸汇订单推仓是三条不同分支。综合看板不读取推送日志；Wayfair 实际提交查 `wf_submission库存变更记录`，HomeDepot 鲸汇库存推送查 `鲸汇库存推送执行结果`，订单选仓查 `鲸汇推仓执行结果_python*`。WF 旧虚拟库存规则、借调分类双事件 / 字段别名和库外推送器读取优先级仍待核实。
- 库龄 / FIFO 补充：普通仓和 CG 的剩余库存、库龄拆分与店铺 FIFO 已确认到 2026-07-14；负剩余库存是欠货表达，不默认清零。统一入库库龄每 4 小时留存，Shipout 支路依赖库外 `ship_out库龄`。过程监控历史停在 2026-07-03，当前 ETL 必须用目标表新鲜度和加总关系验收。
- 销量预测 / 店铺分仓补充：月度填报经 `每日销量预估*` 展开后进入断货预测；08:36 物理表 `销量预估` 只是诊断结果，不是 04:05 普通预测直接输入。三个月销量表保存店铺分子和总分母，下游现场计算比例；CG / CA 被注释店铺的分母缺口与当日守恒 QA 仍待核实。
- 关键风险：预测表当前未见国家仓 / 店铺仓字段；库存拆分字段属于不同拆分轴，不能全部相加；`每月销量售罄预估表` 视图较重，不作为默认高频取数入口；链接 SKU 为 `无` 或空值不能参与链接层聚合。

### 销售 / 组合产品 / DDP

- 通用销售入口：SKU / Model 销量和销售额默认先查 `order_details_tb`；销量用 `数量`，销售额用 `总销售额`，日期优先 `订单日期`。`订单日期` 为空或历史回退场景可参考 `order_details_tb_tj` 的 `订单日期2` 规则，但该视图较重，不作为默认高频入口。
- 销售拆分入口：`总账单详情_extract`、`wf上架sku每日销售额`、三平台上架 SKU 销售映射表。
- 组合产品提醒：`wf上架sku每日销售额.supplier_part` 是销售源 Supplier Part Number / 销售 SKU 层，可为 `Kit`、`Standard`、`Sellable Component` 或少量 `Nonsellable Component`；`总账单详情_extract.上架sku` 是订单拆分子件层，不能直接当同粒度等值关联，也不能把所有 `supplier_part` 都当 `Kit` 父件展开。
- 有效销量提醒：`总账单详情_extract.取消数量` 必须扣减，但组合件场景不能逐组件行直接 `数量 - 取消数量`；OS / HD 样本验证模板为先还原订单上架 SKU 件数，再扣取消数量，最后按组件 / 映射数量扣回。
