# FineBI 业务学习与血缘字段复核流程

## 用途

用于后续学习 FineBI 看板、公共数据、分析主题、自助表、SQL 数据集、指标公式和字段来源。目标是形成可复用的学习闭环，避免只看仪表盘路径、不进节点、不看字段来源、不看公式、不看血缘。

## 适用场景

- 学习新的 FineBI 业务看板。
- 核实某个组件字段的真实来源。
- 追 SQL 数据集、DB 表、自助表右侧流程。
- 确认公式、添加列、关联键、聚合方式和血缘上下游。
- 对报表脚本字段口径做 FineBI 复核。

## 接续学习入口

后续继续学习 FineBI 时，先按轻量索引定位，不要直接全量翻文档或在仪表盘里乱点：

1. 默认先读 `AGENTS.md`、`docs/INDEX.md` 和本 playbook。
2. 需要了解全局已学内容时，读 `docs/summaries/business-summary.md` 中 FineBI 相关摘要。
3. 需要找公共数据未学清单时，读 `docs/finebi/FineBI公共数据目录地图.md` 的“差集学习优先级”。
4. 广告、库存等专项任务，先读对应 summary，再按关键词读原文档相关章节。
5. 进入 FineBI 页面后，优先进入公共数据数据集、分析主题 / 自助表、SQL 数据集或血缘数据表节点；仪表盘只作为入口和下游终点记录。

## 页面类型判断

- 仪表盘 / 看板页：只作为展示入口，记录看板路径、sheet、组件名、筛选器和展示字段。不要把仪表盘画布当成 SQL、公式或字段来源。
- 组件编辑页：用于确认组件字段槽位、维度、指标、结果过滤器、字段来源提示。
- 分析主题 / 自助表编辑页：重点学习右侧流程，包括数据来源、其他表添加列、公式列、汇总列、过滤、分组汇总、字段设置。
- 公共数据 / SQL 数据集页：看数据预览、字段、更新信息、SQL 编辑入口和血缘。
- 血缘视图：确认上游数据表、下游自助表、组件和仪表板；血缘负责指路，公式和 join 仍需进入节点查看。

## 标准学习顺序

1. 先记录看板路径、sheet、组件名和业务问题。
2. 进入组件编辑页，记录维度、指标、结果过滤器。
3. 对关键字段逐个悬停或点击，读取字段提示中的 `来源字段名` 和 `来源表`。
4. 如果字段提示没有出现，不要判定“没有来源”；大概率是悬停位置不对，应移动到字段胶囊文字、字段胶囊边缘、字段列表项或表头字段处重试。
5. 进入来源表，在右侧流程中查该字段。
6. 如果是公式字段，点击公式节点，记录完整公式。
7. 如果是其他表添加列，点击添加列节点，记录来源表、补入字段、聚合方式、匹配键。
8. 如果来源表是 SQL 数据集，进入公共数据或 SQL 编辑页查看 SQL、字段、更新信息。
9. 打开血缘视图，确认上游来源和下游组件 / 看板。
10. 用数据预览、数据库 SQL 或导出样本做 3-5 条抽样复核。
11. 写入对应文档：路径、字段、公式、来源表、join key、粒度、聚合方式、血缘、权限状态、待核实项。

## 血缘视图操作经验

- 血缘节点的数字要点开，数字后面通常还有后续数据表、组件或看板。
- 鼠标悬停节点时肯定会出现可操作提示或跳转入口；如果没有出现，大概率是悬停位置不对。
- 悬停时优先试节点文字、节点图标、节点边框、节点右侧或连接线附近。
- 悬停出现跳转后，进入节点查看表内容、SQL、右侧流程或组件字段。
- 如果跳到仪表盘，只记录它是下游展示终点；追逻辑时要返回主题表、公共数据或 SQL 数据集。
- 血缘只能说明“连到哪里”，不能直接说明“怎么算”。公式、join key、聚合方式必须进入具体节点确认。

## 组件字段来源查看经验

- 在组件编辑页，字段胶囊、字段列表、表头字段都可能出现字段提示。
- 悬停或点击后重点记录：
  - `来源字段名`
  - `来源表`
  - 字段展示名
  - 聚合方式
- 字段提示没有出现时，不要写“页面没有来源信息”；应标注“悬停位置未命中，需换位置重试”。
- 展示字段名不等于真实字段名。例如组件展示 `总销售额`，真实来源可能是 `广告.总销售额修`。
- 看到来源表后必须回来源表继续追，不要停在组件层。

## 右侧流程学习经验

- `其他表添加列`：记录来源表、来源字段、聚合方式、匹配键，是确认 join 的关键。
- `新增公式列`：记录完整公式、输入字段和空值处理。
- `新增汇总列`：记录 DEF / SUM / AVG / COUNTD 等聚合口径和分组字段。
- `过滤`：记录筛选条件，区分全量业务过滤和疑似样例过滤。
- `字段设置`：记录字段重命名、保留字段和字段类型变化。
- `分组汇总`：记录输出粒度，确认是否会改变行数。
- 右侧流程要从当前节点往左侧前置节点和右侧后续节点都看；不要只点当前高亮节点。

## SQL 和 DB 表查看经验

- SQL 数据集优先找 `修改SQL` 或 SQL 编辑入口，只读查看，不保存。
- DB 直连表可能没有 SQL，只能看字段、预览、更新信息和血缘。
- 自助表通常没有完整 SQL，必须拆右侧流程节点看逻辑。
- 如果 SQL 数据集无法复制 SQL，至少记录数据集名、字段、预览样本、更新方式、上游 / 下游血缘和待核实。

## 复核方法

采用三点交叉：

1. 组件侧：字段来源、维度、指标、过滤器。
2. 主题 / 自助表侧：公式、添加列、匹配键、聚合方式。
3. 公共数据 / DB 侧：SQL、物理表字段、数据预览、更新时间。

三点都能对上，才标 `已确认`。只看到一处，标 `待核实`。自己根据名字判断的内容标 `推断`。

## 本项目已验证案例

`WF-WSP广告组广告分析 -> 广告组活动 -> WSP广告组每月汇总` 中展示字段 `总销售额`：

- 组件字段提示：展示字段 `总销售额`，来源字段名 `总销售额修`，来源表 `广告`。
- 主题表公式：`总销售额1 = if(ISNULL(销售额), "", 销售额 / IF(store="WF-FOP",0.93,0.94))`。
- 补列节点：从 `zzdnb_wf上架sku每日销售额` 选择 `销售额`，按求和补入。
- 匹配键：`store = store`、`LEFT(date_est,7) = LEFT(日期,7)`、`sku = wayfair_sku`。
- 血缘：`zzdnb_wf上架sku每日销售额 -> 广告 -> WSP广告组每月汇总 -> WSP广告组广告分析`。
- 结论：最终 WSP 广告组数据表的总销售额相关字段应使用 `zzdnb_wf上架sku每日销售额 / wf上架sku每日销售额.total_revenue`，可按 `store + 月份 + sku=wayfair_sku` 对到广告组和月份。
- 风险：该值是 SKU 月销售额，不是 bid 原生事实；如果按 bid 或日期拆行，不能直接求和，需去重或复用 FineBI `AVG_AGG` 口径。

## 常见坑点

- 悬停不出现提示，通常是位置没悬对，不是节点没有跳转或字段没有来源。
- 仪表盘不是底层逻辑，不要在画布上绕太久。
- 血缘不是公式；血缘只负责找上下游。
- 展示名和来源字段名可能不同。
- 同名字段在不同表里可能是不同口径。
- 比率指标必须按分子分母重算，不能平均明细比率。
- 维度补充字段接到更细粒度后可能重复，汇总前要检查放大。
- 只有 `campaign_name` 时有名称变更风险，广告分析优先使用 `campaign_id`。
