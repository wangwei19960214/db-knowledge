# 广告组月报生成流程

## 适用场景

- Wayfair / WF-WSP 广告组月度经营分析。
- 需要同时看广告表现、广告销售额、自然/总销售、ROI/ROAS、利润、BID、库存有货率、商品状态、评论和评分。
- 适合作为“指定月份 + 指定店铺 + 广告组 / SKU”维度的可复用报表流程。

## 输入参数

- 时间范围：必填，建议按自然月，例如 `2026-01-01` 至 `2026-05-31`。
- 平台 / 店铺：必填，例如 Wayfair、`WF-TD`、`WF-FOP`、`WF-TD-UK`。
- 分析维度：月份、店铺、`campaign_id`、`campaign_name`、平台 SKU、上架 SKU、链接 SKU、采购 SKU、类目、产品状态。
- 输出形式：Excel、CSV、Markdown 摘要、中间 JSON / SQL。
- 核心指标：广告花费、点击、曝光、广告销售额、广告订单数、广告件数、CPC、CTR、CVR、CPM、CPA、ROI、ROAS、利润、成本、总销售额、自然销售额、有货率、BID、评论数、评分。

## 指标口径

- 广告事实主表优先按 `store + campaign_id + sku + 月份` 汇总，`campaign_name` 作为展示字段。
- 若需要历史 bid 效果分析，广告事实主表按 `store + campaign_id + campaign_name + sku + 月份 + product_max_bid` 汇总；`product_max_bid` 是广告底表历史 bid，不是 `Default bid(现在的bid)`。
- `bid/product_max_bid` 同时是展示分组维度；同一广告组 + SKU + 月份存在多 bid 时必须拆行保留，不能先合并后只展示一个 bid。
- 广告销售额默认使用 WSC：`SUM(attributed_wholesale_cost_window_view_through_USD_Day_14)`。
- 广告花费为 `SUM(spend_USD)`。
- 点击、曝光、广告订单数、广告件数均求和；WSP 广告订单数使用 `SUM(attributed_orders_window_view_through_Day_14)`。
- CPC = 广告花费 / 点击。
- CTR = 点击 / 曝光。
- CVR = 广告订单数 / 点击。
- CPM = 广告花费 / 曝光 * 1000。
- CPA = 广告花费 / 广告订单数。
- ROAS = 广告销售额(WSC) / 广告花费。
- ROI = 总利润 / 广告花费；`总利润` 来自 `WF店铺绩效2.0/广告` 上游公式链。
- 利润、成本当前可复用 FineBI `WF广告数据/广告` 字段；公式链为 `总成本 = (每个sku平均成本 * attributed_units_window_view_through_Day_14 / 单买数量) + spend_USD`，`总利润 = WSC * 店铺系数 - 总成本`。单条样本数值复算仍建议作为报表 QA。
- 有货率复用 Wayfair 有货率报表口径，组合产品任一组件可用小于等于 0 则断货。
- BID 需要区分广告日报 `product_max_bid` 与出价表 `Default bid(现在的bid)`、建议 bid。出价表没有 `campaign_id` 字段，但有 `campaignlayer_id`，补当前 bid 快照时优先用 `campaignlayer_id` 对广告日报 `campaign_id`，`Campaign name` 只作展示和兜底。

## 执行步骤

1. 确认时间范围、店铺、输出格式和是否需要利润/库存/评分评论。
2. 搜索已有指标口径、数据表目录、看板映射和坑点。
3. 读取 FineBI 广告专项相关章节，不全文读取全部文档。
4. 查询广告明细并按月、店铺、campaign、SKU 聚合。
5. 取统计期内最新广告组状态，并保留期间状态集合。
6. 关联产品上下架状态、平台 SKU、上架 SKU、链接 SKU、采购 SKU。
7. 关联 BID 表；名称无法匹配时输出未匹配清单。
8. 关联库存有货率、断货天数和异常映射。
9. 关联评论数、评分、滞销标签、类目和利润口径。
10. 汇总并重算比率指标。
11. 输出主表、异常清单、对账清单和说明页。
12. 保存产物到 `reports/`，可复用脚本放 `scripts/`。
13. 更新业务流程、指标、表目录、校验规则、坑点和本 playbook。

## 校验方式

- 广告花费、点击、曝光、广告销售额按月/店铺汇总，对齐 FineBI `WF-WSP整体广告分析` 或 `wf广告`。
- 检查广告、库存、销量、评分评论和利润数据日期覆盖。
- 检查 `campaign_id` 与 `campaign_name` 是否存在多对多或名称变更。
- 检查上架 SKU / 链接 SKU / 采购 SKU 一对多和缺失。
- 检查 WSC 与 retail sales 是否混用。
- 所有比率指标按分子分母重算。
- 利润/成本字段来源标注“待核实”；只有页面明确拒绝访问时才标注“权限不足”。

## 常见坑点

- 广告销售额默认是 WSC，不是 retail sales。
- `campaign_name` 可能变更，不能优先作为主键。
- 利润不是广告日报原生字段。
- `总成本/总利润` 是 FineBI 上游公式链字段，不是广告日报原生字段；公式链已确认，正式报表建议用预聚合小表、带索引临时校验表或 FineBI 已计算样本做数值 QA。
- SKU 映射可能一对多，join 会放大指标。
- BID 表存在 `campaignlayer_id`，补当前 bid 快照时优先对广告日报 `campaign_id`；`Campaign name` 只作展示和兜底。
- 本地 `广告bid设置策略.xlsx` 用作人工策略动作底表时，应先标准化，不覆盖原表；当前口径为向下填充 `campaign_name/model/sku`、忽略源日期、严格按 `store + campaign_name + model + sku` 匹配，且策略只贴 B2C 行。
- 比率指标不能直接平均。
