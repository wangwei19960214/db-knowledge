# 闭环检查清单

每次数据库、FineBI、SQL、报表、指标或业务分析任务结束前，按本清单检查。要求是“按相关性检查并更新”，不是每次机械修改所有文档。

## 任务开始前

- [ ] 已阅读 `AGENTS.md`。
- [ ] 已阅读 `docs/INDEX.md`。
- [ ] 已按入口读取 `.agents/skills/business-data-query/SKILL.md` 或 `.agents/skills/business-data-closed-loop/SKILL.md`。
- [ ] 如涉及问数、学习、报表、SQL、FineBI、指标、排查或沉淀闭环，已阅读 `docs/CLOSURE_PROTOCOL.md`。
- [ ] 已根据需求关键词按需检索相关业务、FineBI、knowledge、lineage 文档。
- [ ] 已确认是否允许连接数据库；默认不连接、不写入。
- [ ] 已确认时间范围、平台、店铺、国家、输出形式和核心指标；缺失时已标注默认假设或待核实。
- [ ] 已判断是否属于快速临时取数 / 迭代改表模式；如果属于，优先复用中间结果并跳过非必要沉淀。

## 执行过程中

- [ ] 未读取或输出本地敏感配置文件内容。
- [ ] 数据库操作保持只读，除非用户明确要求写入或改表。
- [ ] SQL 或脚本已限制必要范围，避免默认全库扫描。
- [ ] 比率指标按分子分母重算。
- [ ] 利润、成本、总销售额等关键字段已说明来源；未确认时标注 `待核实`。
- [ ] 推断内容已标注 `推断`。
- [ ] 权限不足或无法验证事项已标注 `权限不足`。
- [ ] 报表 / Excel / CSV 校验采用最小必要校验，未无故执行大范围 `workbook.inspect`、全表结构 inspect、整表 NDJSON 输出或重型视觉检查。

## 交付前

- [ ] 已生成用户要求的 SQL、脚本、报表、分析结论或文档。
- [ ] 已记录产物路径。
- [ ] 已说明关键口径、过滤条件和校验方式。
- [ ] 已说明异常数据、坑点、权限限制或未验证风险。
- [ ] 如有正式报表产物，已更新或检查 `reports/INDEX.md`；临时 / 迭代文件已说明跳过索引原因。
- [ ] 已清理非业务临时文件，包括 `inspect.ndjson`、`*.xlsx.inspect.ndjson`、大型预览日志、临时 `node_modules` 链接和无必要预览图。
- [ ] `reports/INDEX.md` 未索引 inspect / sidecar / 预览图 / 临时校验日志，除非用户明确要求。

## 文档闭环路由

文档路由以 `docs/CLOSURE_PROTOCOL.md` 为唯一共享协议来源。本清单只提醒执行检查，不维护另一套完整路由。

## 最终回复应包含

按 `docs/CLOSURE_PROTOCOL.md` 输出 Closure Packet；面向用户可压缩展示，但不能省略产物路径、校验结果、待核实项和文档更新结论。
