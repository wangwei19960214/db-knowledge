# 智能取数安全边界

本文件规定智能取数相关任务的安全边界。除非用户明确要求写入、改表或执行会改变数据的过程，默认只读。

## 数据库操作边界

- 默认只允许查询类操作。
- 禁止在未获明确授权时执行写入、改表、删除、清空、导入、触发任务等会改变数据的操作。
- 禁止启动长期占用数据库连接的后台进程。
- 查询结束后必须关闭连接或连接池。
- 遇到连接超时、连接数异常、白名单问题时，先暂停新增数据库任务，排查连接状态。

## 敏感信息边界

- 不读取、不展示、不记录任何本地连接配置文件内容。
- 不在回复、文档、日志、脚本输出中写入数据库凭据、连接串、密钥或访问令牌。
- 需要说明连接问题时，只描述现象和排查动作，不复述敏感配置。
- 文档里只记录表、字段、口径、脚本路径和产物路径。

## 口径安全边界

- 不编造表、字段、SQL、指标口径、FineBI 血缘。
- 未确认信息标注 `待核实`。
- 根据文件名或上下文推断的信息标注 `推断`。
- 权限不足导致无法验证的信息标注 `权限不足`。
- 广告销售额必须区分 WSC / wholesale cost 与 retail sales。
- ROAS、ROI、CTR、CVR、CPA、CPM、CPC 等比率指标必须在汇总层按分子分母重算。
- Campaign 分析优先使用 `campaign_id`。
- 利润、成本、总利润、总成本必须说明字段来源；来源未确认时标注 `待核实`。

## 数据量边界

- 新需求第一次查询应优先小范围验证，例如限制时间、店铺、SKU 或行数。
- 大范围取数前，应确认业务目标、时间范围、输出形式和是否需要落地报表。
- 不把全库扫描当作默认动作；优先按业务问题定向取数。

## 交付边界

- SQL、脚本、报表和分析结论必须能追溯到文档、脚本或产物路径。
- 每次需求结束前，按 `docs/CLOSED_LOOP_CHECKLIST.md` 检查是否需要更新知识卡片、指标字典、表目录、血缘、待核实池、报表索引。
- 闭环要求是“按相关性检查并更新”，不是机械修改所有文档。
