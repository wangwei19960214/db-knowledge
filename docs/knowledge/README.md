# 智能取数知识体系说明

本目录用于沉淀“公司智能取数”的可复用知识。它不是数据库全量字典，也不替代业务需求文档；它负责把已经验证或高价值待核实的取数经验整理成机器和人都容易查找的结构。

## 使用方式

后续遇到取数、报表、FineBI 复核、异常排查需求时，先按下面顺序判断：

1. 看用户问题属于哪个取数意图。
2. 查 `取数意图字典.md` 和 `问法到指标映射.md`，确定主题、指标和风险。
3. 查 `数据知识沉淀表.md`，复用已有知识卡片。
4. 需要写 SQL 时，再查 `SQL模板库.md`。
5. 涉及跨表链路时，查 `业务实体关系图.md` 和 `docs/lineage/`。
6. 结束前按 `docs/CLOSED_LOOP_CHECKLIST.md` 做闭环检查。

## 文件分工

| 文件 | 用途 |
| --- | --- |
| `数据知识沉淀表.md` | 一张张知识卡片，记录业务场景、表、口径、脚本、产物和坑点。 |
| `取数意图字典.md` | 把用户自然语言需求归类成可执行的取数意图。 |
| `SQL模板库.md` | 保存可复用 SQL 骨架；不确定字段统一标 `TODO: 待核实`。 |
| `业务实体关系图.md` | 记录 SKU、Model、店铺、广告、库存、销售、FineBI 等实体关系。 |
| `问法到指标映射.md` | 把常见问法映射到指标、维度、默认口径和风险提示。 |
| `智能取数安全边界.md` | 约束只读、权限、敏感信息、数据量和口径安全。 |

## 状态标记

- `已验证`：来自已有需求沉淀、脚本、报表或 FineBI 梳理，且已有明确口径。
- `待核实`：已有线索但缺少字段、来源、刷新逻辑或业务确认。
- `推断`：根据文件名、已有链路或上下文推断，尚未被数据库或业务确认。
- `权限不足`：需要访问数据库、FineBI 或外部系统但当前无法验证。

## 更新规则

- 每完成一次取数或报表需求，优先追加到 `数据知识沉淀表.md`。
- 新意图、新问法或新指标映射，追加到 `取数意图字典.md` 或 `问法到指标映射.md`。
- 新链路、新结果表、新刷新任务，追加到 `docs/lineage/`。
- 不把凭据、连接串或敏感输出写入本目录。
