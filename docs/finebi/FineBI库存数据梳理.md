最近追加日期：2026-06-26

## 范围

- FineBI 地址：`http://192.168.110.18:37799/webroot/decision`
- 公共数据负责人目录：王威、tsq、章森淼
- 本轮重点：库存、仓库可用、预计到仓、断货、库存销量、CG 仓库存
- 数据连接：目前核实的核心 SQL 均使用 `zzdnb`

## 核心结论

1. 王威维护实时库存与预计到仓的核心 SQL，底层来自 `warehouse_stock_最新`、`公司sku上下架表`、`预计到仓表`。
2. 该核心 SQL 不仅用于王威自己的库存看板，也被 tsq 的“多联动看板”和“上架产品半年内看板”复用。
3. 章森淼维护仓库可用预估和断货预测链路，核心事实表是 `tb_kyxl_forecast_and_reality`，FineBI 入口 `数据连接_tb_kyxl_forecast_and_reality` 当前有 867,763 行、15 个字段，另有合并 SKU 版本 `tb_kyxl_forecast_and_reality_合并sku`。
4. `采购sku_model映射_2` 是库存预估的重要公共映射层，同时服务普通 SKU 和合并 SKU 两套看板。
5. CG 仓库存单独从 `wayfair后台cg可用` 取最新日期数据，再关联 `cg_part_sku映射`。
6. “每天各平台各仓库可用”看板的核心来源不是 `warehouse_stock_最新`，而是数据库历史表 `库存拆分记录`。
7. 国家库存数据集均以 `仓库映射表.所在国家` 过滤仓库；美国版还计算美国两个 CG 仓的锁定库存。
8. Shipout 仓储费链路使用 `shipout每天库存` 按日期、SKU、库龄汇总；FineBI 中对应的数据库数据集名为 `zzdnb_shipout每天库存_2`。
9. CG 仓各仓库存分析由 `zzdnb_cg仓每天库存` 提供每日仓库库存历史，由 `zzdnb_cg仓库库存借调` 提供借调数量，二者同属 `库存拆分相关更新任务`。
10. 库存拆分宽表 `产品库存拆分` 有 20 个字段，统一承载系统库存、其他仓/CG 仓、AMZ/垂直渠道及出入库拆分。
11. 总销量售罄主表 `每月销量售罄预估表` 有 28 个字段，结合可用、在途、计划发货、CG 锁定和未来三个月销量计算售罄与断货。
12. `各sku每月断货天数` 是按月 SKU 粒度的断货明细 SQL 层，断货日来自 `可用=0` 或 `预估可用=0`，断货数量同时考虑未来预估日均销量和已发生断货的三个月不断货销量均值。

### 各 SKU 每月断货天数

- 所属：`公共数据/章森淼/仓库库存可用预估`
- 数据集：`各sku每月断货天数`
- 类型：SQL 数据集
- 数据连接：`zzdnb`
- 数据量：`22,006` 行

字段口径：

- `年月`、`sku`：按月、SKU 聚合。
- `断货天数`：当月 `可用=0` 或 `预估可用=0` 的天数。
- `断货开始日期`、`断货结束日期`：当月断货日的最小/最大日期。
- `断货数量_1`：未来或预估可用为 0 时，用 `日均销量` 估算断货损失。
- `断货数量_3`：已发生断货时，用最近三个月不断货销量 / 不断货天数 * 已发生断货天数估算损失。
- `断货数量`：当前月及历史月取 `断货数量_1 + 断货数量_3`；未来月只取 `断货数量_1`。
- `有货率`：`不断货销量 / (不断货销量 + 断货数量)`。

SQL：

```sql
select a.*,
  case
    when 年月 <= DATE_FORMAT(CURRENT_DATE,"%Y-%m")
    then 断货数量_1 + 断货数量_3
    else 断货数量_1
  end as 断货数量,
  ifnull(
    round(
      不断货销量 / (
        不断货销量 +
        case
          when 年月 <= DATE_FORMAT(CURRENT_DATE,"%Y-%m")
          then 断货数量_1 + 断货数量_3
          else 断货数量_1
        end
      ),
      4
    ),
    0
  ) as 有货率
from (
  SELECT *,
    SUM(已发生不断货月销量) OVER (
      PARTITION BY sku
      ORDER BY 年月
      ROWS BETWEEN 2 PRECEDING AND CURRENT ROW
    ) AS three_month_total_sales,
    SUM(已发生不断货天数) OVER (
      PARTITION BY sku
      ORDER BY 年月
      ROWS BETWEEN 2 PRECEDING AND CURRENT ROW
    ) AS three_month_bdh_days,
    ifnull(
      round(
        SUM(已发生不断货月销量) OVER (
          PARTITION BY sku
          ORDER BY 年月
          ROWS BETWEEN 2 PRECEDING AND CURRENT ROW
        )
        / SUM(已发生不断货天数) OVER (
          PARTITION BY sku
          ORDER BY 年月
          ROWS BETWEEN 2 PRECEDING AND CURRENT ROW
        )
        * 已发生断货天数,
        0
      ),
      0
    ) as 断货数量_3
  from (
    SELECT
      DATE_FORMAT(date,"%Y-%m") as 年月,
      sku,
      sum(case when 可用=0 or `预估可用`=0 then 1 else 0 end) as 断货天数,
      min(case when 可用=0 or `预估可用`=0 then date else null end) as 断货开始日期,
      max(case when 可用=0 or `预估可用`=0 then date else null end) as 断货结束日期,
      ifnull(round(sum(case when `预估可用`=0 then `日均销量` else 0 end),0),0) as 断货数量_1,
      sum(case when `可用`!=0 then `销量` else 0 end) as 已发生不断货月销量,
      sum(case when `可用`!=0 then 1 else 0 end) as 已发生不断货天数,
      sum(case when `可用`=0 then 1 else 0 end) as 已发生断货天数,
      ifnull(
        round(
          sum(case when `可用`!=0 then `销量` else 0 end)
          / sum(case when `可用`!=0 then 1 else 0 end)
          * sum(case when `可用`=0 then 1 else 0 end),
          0
        ),
        0
      ) as 断货数量_2,
      round(
        sum(
          case
            when `可用`!=0 and 预估可用 is null then `销量`
            when `预估可用`!=0 and 可用 is null then `日均销量`
            else 0
          end
        ),
        0
      ) as 不断货销量,
      round(sum(case when `销量` is null then `日均销量` else 销量 end),0) as 本月销量
    from tb_kyxl_forecast_and_reality
    GROUP BY DATE_FORMAT(date,"%Y-%m"),sku
  ) as a
) as a
left join `考虑放弃sku清单` as b
  on a.sku = b.sku
where b.sku is null
```

## 3. 采购 SKU / Model 映射

### 公共数据集

- 所属：`公共数据/章森淼/仓库库存可用预估`
- 数据集：`采购sku_model映射_2`
- 数据连接：`zzdnb`

### 数据库底表

- `平台产品详情表`
- `tb_kyxl_forecast_and_reality`
- `公司sku上下架表`
- `仓库每日销量断货情况_表`

### 关联逻辑

- 用递归 CTE 拆分 `平台产品详情表.采购sku` 中以逗号分隔的多个采购 SKU。
- 按采购 SKU 聚合 Model。
- 补入预测事实表和公司上下架表中存在、但平台产品详情表中可能缺失的 SKU。
- 通过 `公司sku上下架表` 补充分类。
- 通过 `仓库每日销量断货情况_表` 补充“是否新品”。

### 应用位置

- `zsm的分析/zzd/仓库库存可用预估`
- `zsm的分析/zzd/仓库库存可用预估合并sku`

### SQL

```sql
SELECT
  b.采购sku,
  ifnull(SUBSTRING_INDEX(category,"-",1),"未归类") as first_class,
  ifnull(SUBSTRING_INDEX(category,"-",-1),"未归类") as second_class,
  b.model,
  case
    when SUBSTRING_INDEX(category,"-",1)="分销"
    then SUBSTRING_INDEX(category,"-",1)
    else c.是否新品
  end as 是否新品
FROM
(
  SELECT 采购sku,GROUP_CONCAT(distinct model) as model,first_class,second_class
  from
  (
    (
      WITH RECURSIVE t(产品id,model,first_class,second_class,`采购sku`,str) AS (
        SELECT 产品id,model,first_class,second_class,
               substr(`采购sku`,1,instr(`采购sku`,',')-1),
               substr(concat(采购sku,','),instr(`采购sku`,',')+1)
        FROM 平台产品详情表 t
        where length(采购sku)>0
        UNION ALL
        SELECT 产品id,model,first_class,second_class,
               substr(str,1,instr(str,',')-1),
               substr(str,instr(str,',')+1)
        FROM t
        WHERE instr(str,',')>0
      )
      SELECT 产品id,model,first_class,second_class,采购sku
      FROM t
      WHERE length(采购sku)>0 and LENGTH(model)>0
    )
    union
    select null,null,null,null,sku
    from tb_kyxl_forecast_and_reality
    GROUP BY sku
    union
    select null,null,null,null,sku
    from `公司sku上下架表`
    GROUP BY sku
  ) as a
  group by 采购sku
) as b
left join `公司sku上下架表` as a
  on a.sku=b.`采购SKU`
left join (
  select sku,是否新品
  from `仓库每日销量断货情况_表`
  GROUP BY sku
) as c
  on a.sku=c.sku
```

## 11. Wayfair 上架 SKU 与链接 SKU 有货率

### 已核实数据源

- 在架商品：`平台产品详情表`
- 专用销售映射：`wayfair上架sku销售映射`
- 库存历史：`warehouse_stock_在售`
- Wayfair 销量：`wf_supplier_parts_performance`
- 广告记录：`advertising_product_report_by_day`
- 国家仓库范围：`仓库映射表`

### 关联与计算口径

- 上架 SKU 粒度为“店铺 + 上架 SKU”，按上架日期、下架日期逐日判断是否在架；上架日计入，下架日不计入。
- 采购 SKU 优先使用 `wayfair上架sku销售映射`，缺失时用 `平台产品详情表.采购sku` 拆分补缺。
- 库存按采购 SKU、日期、店铺对应国家仓库汇总 `可用`。当天无记录时向前取最近有效库存，此前从未出现则标记缺失。
- 美国店铺 `WF-TD`、`WF-FOP` 使用美国仓；`WF-TD-CA`、`WF-FOP-CA` 使用加拿大仓；`WF-TD-UK` 使用英国仓。
- 组合产品任一采购 SKU 当天可用小于等于 0，即判定该上架 SKU 当天断货；全部组件大于 0 才算有货。
- 上架 SKU 有货率为“有货天数 / 实际在架天数”。存在缺失库存日时，严格有货率留空，另提供已知日有货率用于排查。
- 链接 SKU 按“店铺 + 链接 SKU”聚合。空值、`NULL`、`无` 不作为有效链接。
- 每个上架 SKU 的销量权重为截至统计截止日向前最近 90 个有货日的平均销量，即“销量合计 / 有货日数”。
- 链接 SKU 每日有货率为 `Σ(上架SKU销量权重 × 当天有货状态) / Σ(当天实际在架上架SKU销量权重)`。
- 某链接当天全部在架上架 SKU 的有效销量权重总和为 0 时，使用等权回退。
- 广告通过“店铺 + 平台 SKU”关联，输出统计期内全部广告 ID 和广告名称。

### 2026-06-25 SKU / Model / 上架 SKU 映射补充

- `wayfair上架sku销售映射.part_number` 对应 `平台产品详情表.上架sku`，不是 `平台sku` 或 `链接sku`；后续有货率 SQL 应按 `store + 上架sku = store + part_number` 关联。
- `wayfair上架sku销售映射` 物理主键为 `store + part_number + 采购sku`；本轮只读确认存在多组件上架 SKU，`3,736` 个 `store + part_number` 对应多个采购 SKU，最大组件数 `5`。
- `平台产品详情表.采购sku` 可能为空或含逗号分隔多采购 SKU；有货率链路中缺少专用映射时可以回退该字段，但必须拆分并记录缺映射风险。
- `采购sku_model映射表` 是当前业务库可查的采购 SKU -> Model 物理表；FineBI `采购sku_model映射_2` 仍按数据集线索处理，和物理表的差异待核实。
