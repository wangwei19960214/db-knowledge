# 看板到数据集映射

## 使用原则

- 本文件只记录已在现有文档中出现的 FineBI 看板、组件、数据集和血缘线索。
- 未进入看板或未确认 SQL 的内容标注“待核实”。
- 血缘冲突或权限不足时保留说明，不把推断写成事实。

## WF-WSP整体广告分析

- FineBI 路径：`zzd/广告`
- 业务用途：按月、店铺查看 WF/WSP 广告、自然销售和总销售整体表现。
- 主要筛选器：`年月区间`、`店铺`
- 分析粒度：月度店铺粒度，`store + 年月`
- 使用数据集：`公共数据/王威/广告/总广告数据`；上游含 `WF广告汇总`；是否经过自助数据集二次计算待核实。

- 关键维度：店铺、年月。
- 已确认公式：比率指标应按分子分母重算；具体组件公式待核实。
- 组件/字段：`广告vs自然销售额`、`广告增长VS销售增长`、`广告CR与自然CR百分比对比（*100）`、`广告花费、销售额、利润与ROI分析`、`广告花费占比`、`广告点击转化`、`广告曝光点击`。
- 血缘关系：2026-06-18 复核确认 `总广告数据 -> 总广告数据_广告组数据` 右侧数字 `109`，`总广告数据 -> 总广告数据` 右侧数字 `14`；继续展开后确认 7 个整体组件均进入 `WSP广告分析`：`广告CR与自然CR百分比对比（*100）`、`广告vs自然销售额`、`广告曝光点击`、`广告花费占比`、`广告花费、销售额、利润与ROI分析`、`广告点击转化`、`广告增长VS销售增长`。`总广告数据_广告组数据` 进入广告组主题链路。

## WF-WSP广告组广告分析

- FineBI 路径：`zzd/广告`
- 业务用途：按广告组、店铺、SKU、分类和标签分析广告投放、利润、库存可用和商品质量。
- 主要筛选器：`日期区间`、`广告组`、`店铺`、`活动垂直标签`、`isB2B`、`wayfair_sku`
- 分析粒度：广告组月度组件常见粒度为 `store + 年月 + campaign_name + sku/model`；精确粒度待核实。
- 使用数据集：`总广告数据_广告组数据`、`分类目广告数据_广告组数据`、`WF广告数据/广告`；已见上游/辅助数据集包括 `zzdnb_wf上架sku每日销售额`、`wf_sku不断货销量`、`zzdnb_detailed_listing_health`、WF 广告日报、商品辅助标签和 bid 数据集。
- 核心指标：CPA、CVR、ROI、曝光、点击、花费、评论数、评分、利润、CPM、CTR、断货百分比、广告销售额、广告订单数、广告销售个数、总销售额、广告销售占比、成本、CPC、ROAS。
- 关键维度：`store`、`日期(年月)`、`model`、`campaign_name`、`标签`、`sku`、`isB2b`、`活动是否有ddp`、`class_name`、`分类`、`campaign_status`、`first_10_part_numbers`、`product_status`、`bid`、`广告组利润类型`、`是否有价格优势`。
- 已确认公式：`广告销售额 = SUM_AGG(attributed_wholesale_cost_window_view_through_USD_Day_14)`；`成本 = SUM_AGG(总成本)`；`利润 = SUM_AGG(总利润)`；`广告ROI = SUM_AGG(总利润) / SUM_AGG(spend_USD)`；`广告ROAS = 广告销售额(WSC) / SUM_AGG(spend_USD)`；`断货百分比 = SUM_AGG(断货part个数) / SUM_AGG(活动的上架sku数量)`；`总销售额修 = IF(AVG_AGG(总销售额1)=0,"",AVG_AGG(总销售额1))`；`广告销售占比 = 广告销售额 / AVG_AGG(总销售额1)`；`广告花费占广告销售额的比 = SUM_AGG(spend_USD) / SUM_AGG(WSC)`；`广告组利润类型` 按总利润正负分类。
