### FineBI 广告占比与断货百分比补充学习

- 提出时间：2026-06-17
- 业务目标：使用 `business-data-closed-loop` 流程继续学习 FineBI 广告相关口径，只按关键词读取摘要和原文档相关章节，并补齐广告组组件中部分待核实占比公式。
- 输出形式：更新业务语义层和 FineBI 广告专项文档。
- 涉及维度：店铺、年月、广告组、SKU、isB2B、DDP 状态、分类、campaign。
- 核心指标：`断货百分比`、`广告销售占比`、`广告花费占广告销售额的比`、`bid`。
- 使用表/视图/过程：
  - FineBI `wangwei的分析/广告/WF广告数据/广告`
  - FineBI `wangwei的分析/广告/WF广告数据/WSP广告组每月汇总`
  - `公共数据/王威/增量更新/wf广告`
  - `公共数据/tsq/WF店铺曝光/zzdnb_wf广告出价表`（仍待完成组件级对齐）
- 指标口径：
  - `断货part百分比 = IF(SUM_AGG(断货part个数)=0,"",SUM_AGG(断货part个数)/SUM_AGG(活动的上架sku数量))`。
  - `广告销售占比 = IF(OR(广告销售额=0,总销售额修=0),"",广告销售额/AVG_AGG(总销售额1))`。
  - `广告花费占广告销售额的比 = SUM_AGG(spend_USD)/SUM_AGG(attributed_wholesale_cost_window_view_through_USD_Day_14)`。
- 验证方式：Chrome 进入 `WF广告数据/WSP广告组每月汇总` 组件，展开 `广告` 表字段列表，按关键词搜索并打开计算字段公式弹窗读取。
- 异常数据和坑点：
  - `广告销售占比` 公式分母使用 `总销售额1`，条件判断使用 `总销售额修`；该关系已在 2026-06-17 后续学习中确认：`总销售额修 = IF(AVG_AGG(总销售额1)=0,"",AVG_AGG(总销售额1))`。
  - `bid` 组件维度未直接弹出来源提示；`广告` 表字段可见 `Default bid(现在的bid)`、`Suggested Bid detail(具体金额)`、`最小建议bid`、`最大建议bid`，仍需和 `zzdnb_wf广告出价表` 做组件级对齐。
  - 本轮尝试搜索 `OS广告` 看板未成功切换，OS 实际使用关系仍待核实。
- 最终产物路径：
  - `docs/finebi/FineBI广告数据梳理.md`
  - `docs/business/指标口径字典.md`
  - `docs/business/看板到数据集映射.md`
  - `docs/business/校验规则.md`
  - `docs/business/已知坑点.md`
  - `docs/summaries/finebi-ad-summary.md`
  - `docs/summaries/business-summary.md`
- 后续可复用点：后续广告组月报或对账时，断货百分比、广告销售占比和广告花费/WSC 比例均应按本轮确认的分子分母重算。

### FineBI 广告利润口径梳理

- 提出时间：2026-06-16
- 业务目标：继续学习并梳理 FineBI 广告看板利润部分，重点覆盖广告利润、总成本、广告销售额、自然销售额、采购 SKU 维度利润和利润率。
- 输出形式：更新专项文档。
- 涉及维度：店铺、年月、广告组、SKU、采购 SKU、分类、产品状态、B2B、DDP 状态。
- 核心指标：`利润`、`总利润`、`成本`、`总成本`、`广告销售额`、`自然销售额`、`广告ROI`、`广告ROAS`、`广告组利润类型`。
- 使用表/视图/过程：
  - FineBI `wangwei的分析/广告/WF广告数据/广告`
  - FineBI `wangwei的分析/广告/WF广告数据/WSP广告组每月汇总`
  - `公共数据/王威/增量更新/wf广告`
  - `总广告数据_广告组数据`
  - `分类目广告数据_广告组数据`
- 关键 SQL 或脚本路径：详见 `FineBI广告数据梳理.md` 中 `公共数据/王威/增量更新/wf广告` SQL。
- 指标口径：
  - `广告销售额 = SUM_AGG(attributed_wholesale_cost_window_view_through_USD_Day_14)`，为 0 时显示空。
  - `成本 = SUM_AGG(总成本)`，为 0 时显示空。
  - `利润 = SUM_AGG(总利润)`，为 0 时显示空。
  - `广告ROI = SUM_AGG(总利润) / SUM_AGG(spend_USD)`，分母或利润为 0 时显示空。
  - `广告ROAS = SUM_AGG(attributed_wholesale_cost_window_view_through_USD_Day_14) / SUM_AGG(spend_USD)`，分母或广告销售额为 0 时显示空。
  - `广告组利润类型` 按 `总利润` 正负分为 `盈利广告组`、`亏损广告组`、`无效广告组`。
- 验证方式：在 Chrome 中进入 FineBI `WF广告数据` 主题，打开 `广告` 表字段和 `WSP广告组每月汇总` 组件，读取计算字段公式；进入公共数据 `wf广告` SQL 编辑确认采购 SKU 映射 SQL。
- 异常数据和坑点：
  - `广告销售额` 采用 WSC 字段，不是零售额字段。
  - `WF广告数据/广告` 的基础源显示为 `wangwei的分析/店铺绩效/WF店铺绩效2.0/广告`；此前曾记录该节点提示 `当前表无权限`，但 2026-06-17 用户确认当前权限足够，`总成本/总利润` 底层生成需继续进入该上游验证。
  - `采购SKU` 在公共 SQL 和自助表字段中已确认存在，但当前已见 `WSP广告组每月汇总` 组件维度没有直接放出 `采购SKU`。
- 最终产物路径：`FineBI广告数据梳理.md`
- 后续可复用点：后续广告组利润报表可直接复用本次确认的 `利润/成本/ROI/ROAS` 公式，并继续补齐 `WF店铺绩效2.0/广告` 的成本利润生成逻辑。

### Wayfair 2026年1-5月广告组库存综合分析

- 提出时间：2026-06-16
- 业务目标：输出 Wayfair 平台 2026 年 1-5 月广告组分析数据，结合广告表现、库存有货率、链接/产品状态、滞销标签、类目、评论评分、利润和总销售表现，服务广告投放分析。
- 输出形式：计划输出 Excel 报表及中间 JSON/SQL 结果，建议放在 `reports/wayfair_ad_group_analysis_2026-01_2026-05/`。
- 时间范围：2026-01-01 至 2026-05-31。
- 涉及维度：月份、店铺、广告组/campaign、平台 SKU、上架 SKU、链接 SKU、采购 SKU、类目、产品状态、广告组状态。
- 核心指标：有货率、广告组状态、产品是否在售、滞销标签、类目、平台 SKU、CPA、CPM、CTR、CVR、利润、BID、CPC、评论数、评分、广告销售额、总销售额、广告销售占比、广告花费、点击、曝光、订单量、销量。
- 使用表/视图/过程：
  - `advertising_product_report_by_day`
  - `wf广告出价表`
  - `平台产品详情表`
  - `wayfair上架sku销售映射`
  - `warehouse_stock_在售`
  - `仓库映射表`
  - `wf_supplier_parts_performance`
  - `产品库存标签`
  - `产品销量及评论评分`
  - `上架sku利润表`
  - `广告组合单买详情`
- 关键 SQL 或脚本路径：
  - 待新增：`scripts/generate-wayfair-ad-group-analysis-data.js`
  - 历史待新增：`scripts/build-wayfair-ad-group-analysis-workbook.mjs`（2026-06-29 Phase 1 核验：当前磁盘未找到；已存在 `scripts/build-wayfair-ad-group-analysis-workbook.py`，是否完全替代原计划 `.mjs` 待确认。）
  - 可复用参考：`scripts/generate-wayfair-availability-data.js`
- 指标口径：
  - 广告事实主表优先使用 `advertising_product_report_by_day`，按 `store + campaign_id/campaign_name + sku + 月份` 汇总。
  - 广告组状态取统计期内最新日期的 `campaign_status`、`campaign_is_active`，同时保留期间状态集合。
  - 产品是否在售优先来自 `平台产品详情表` 的上下架日期和 `上下架状态`；广告日报 `product_status` 作为平台广告侧状态补充。
  - 有货率复用 Wayfair 上架 SKU 与链接 SKU 有货率口径：店铺映射国家仓，采购 SKU 逐日查 `warehouse_stock_在售.可用`，组合产品任一组件可用小于等于 0 则当天断货。
  - CPA、CPM、CTR、CVR、CPC 在汇总后按分子分母重算，不平均明细比率。
  - BID 同时考虑 `advertising_product_report_by_day.product_max_bid` 和 `wf广告出价表.Default bid(现在的bid)`，最终以报表字段区分。
  - 评论数、评分来自 `产品销量及评论评分` 的 `Review_Count`、`Review_Rating`。
  - 滞销标签来自 `产品库存标签.是否为滞销品`，可同步输出积压、缺货、重点品等辅助标签。
  - 利润先用 `上架sku利润表` 建立商品/月利润口径；广告归因利润如需精确，需要进一步确认广告销售分摊方式。
- 验证方式：
  - 核对 2026-01 至 2026-05 核心表日期覆盖。
  - 将广告点击、曝光、花费、广告销售额按月/店铺汇总后，对照 FineBI `WF-WSP整体广告分析` 或 `wf广告` 看板。
  - 抽样核对平台 SKU 到上架 SKU/采购 SKU/库存国家仓映射。
- 异常数据和坑点：
  - 同一平台 SKU 可能存在多个上架 SKU、链接 SKU 或采购 SKU，需要保留异常清单。
  - 广告组名称可能变更，建议以 `campaign_id` 为主键、`campaign_name` 为展示字段；若出价表只有名称，则需做名称回填并标注未匹配。
  - 广告销售额存在零售额和批发成本两种字段，分析时需明确采用哪一个作为主销售额。
  - 利润字段当前不是广告日报原生字段，需单独对齐商品利润或分摊口径。
- 最终产物路径：待生成。
- 后续可复用点：可沉淀为 Wayfair 广告组月度经营分析模板，后续仅改时间范围即可复跑。

### Wayfair WSP 广告报表字段来源与实现粒度复核

- 提出时间：2026-06-18
