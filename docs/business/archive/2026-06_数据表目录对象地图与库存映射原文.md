### 关键字段和关联关系

- Wayfair 上架 SKU 有货率：`平台产品详情表` 定义在架范围；`wayfair上架sku销售映射` 优先补采购 SKU 和数量；缺失时回退 `平台产品详情表.采购sku`；再按采购 SKU、日期、国家仓在 `warehouse_stock_在售` 汇总 `可用`。
- 国家仓范围：`WF-TD`、`WF-FOP` 走美国仓；`WF-TD-CA`、`WF-FOP-CA` 走加拿大仓；`WF-TD-UK` 走英国仓；仓库归属依赖 `仓库映射表.后台仓库名称 -> 所在国家`，关联前需按 `后台仓库名称` 去重。
- 组合 SKU：任一组件采购 SKU 当天可用小于等于 0，判定该上架 SKU 当天断货；全部组件大于 0 才算有货。
- 断货预测：`tb_kyxl_forecast_and_reality` 的 `可用 = 0` 或 `预估可用 = 0` 计为断货日；`各sku每月断货天数` 按月聚合并排除 `考虑放弃sku清单`。
- CG 仓库存：`wayfair后台cg可用.part_number = cg_part_sku映射.上架sku` 且 `store` 相等；库存取最新 `create_time` 日期。

### 常见问法到取数思路

| 用户问法 | 优先对象 | 取数思路 |
| --- | --- | --- |
| 某个 SKU 现在还有多少库存？ | `warehouse_stock_在售` / `warehouse_stock_最新` | 先确认要历史日还是实时库存；历史用 `warehouse_stock_在售`，实时 FineBI 口径看 `warehouse_stock_最新`。 |
| Wayfair 某店铺上架 SKU 有货率是多少？ | `平台产品详情表`、`wayfair上架sku销售映射`、`warehouse_stock_在售`、`仓库映射表` | 先生成在架日历，再按国家仓汇总采购 SKU 可用，组合 SKU 按最小组件可用判断。 |
| 链接 SKU 有货率是多少？ | 上架 SKU 有货率链路 + 链接 SKU 字段 | 先算上架 SKU 日有货状态，再按链接 SKU 和销量权重聚合；权重缺失时可等权回退。 |
| 哪些 SKU 有断货风险？ | `tb_kyxl_forecast_and_reality`、`考虑放弃sku清单` | 查未来日期 `可用` / `预估可用` 为 0 的 SKU，排除放弃清单。 |
| 某月断货 SKU 数量或有货率？ | `各sku每月断货天数` / `zzdnb_断货sku数量` | 分 SKU 明细用 `各sku每月断货天数`；整体 KPI 用 `zzdnb_断货sku数量`。 |
| CG 仓库存是多少？ | `wayfair后台cg可用`、`cg_part_sku映射` | 取最新日期，按店铺和 part number 补采购 SKU，仓库名拼为 CG 仓。 |

### 已知坑点

- `warehouse_stock_在售`、`warehouse_stock_最新`、`库存拆分记录` 的适用场景不同，不能混用为同一张库存主表。
- 上架 SKU、链接 SKU、采购 SKU 可能一对多或缺失；有货率必须先处理映射和组合数量。
- 库存、销量、广告数据截止日期可能不同；有货率报表应输出各来源截止日期。
- 有货率分母是实际在架天数或销量权重，不是自然日简单平均；缺失库存日需要单独标记。
- 月度 KPI 表适合看汇总，不适合直接追分日断货；分日核对应回 `tb_kyxl_forecast_and_reality`。
- `垂直采购sku销量断货情况表`、`os库存变更记录` 当前仅为高优先级线索，未确认字段前不能直接作为主事实表。

6. `垂直采购sku销量断货情况表`、`os库存变更记录` 是否是可复用结果表，还是历史 / 中间表。

### 智能取数对象识别顺序

1. 先判断输入是否命中 `采购sku_model映射表.model`；若命中，按 Model 展开到采购 SKU，再进入库存、销售或断货链路。
2. 再判断是否命中 `采购sku_model映射表.采购sku` 或库存事实表的采购 SKU 口径。
3. Wayfair 场景判断 `平台产品详情表` 的 `上架sku`、`平台sku`、`链接sku`、`model`、`采购sku`，但后续补组件时优先用 `wayfair上架sku销售映射`。
4. 当需要 Wayfair 上架 SKU 到采购 SKU 组件时，按 `store + 上架sku = store + part_number` 关联 `wayfair上架sku销售映射`。
5. 若 `wayfair上架sku销售映射` 缺失，则回退 `平台产品详情表.采购sku`；若该字段含逗号，必须拆分后再查采购 SKU 级事实表。
6. 查询库存时，最终必须落到采购 SKU，再查 `warehouse_stock_在售.sku` 或实时库存相关表；不要直接用 Model、平台 SKU、链接 SKU 查库存主表。
