# 指标口径字典

## 使用原则

- 本文件沉淀第一版业务语义层，后续按任务追加，不静默覆盖历史。
- 未确认内容标注“待核实”；推断内容标注“推断”。
- 广告销售额必须区分 WSC / wholesale cost 与 retail sales。
- ROI、ROAS、CTR、CVR、CPA、CPM、CPC 等比率指标在汇总层必须按分子分母重算，不能直接平均明细比率。
- Campaign 分析优先使用 `campaign_id`；只有 `campaign_name` 时必须提示名称变更风险。

## 广告销售额

- 业务含义：广告归因销售额；在已确认 WF 广告利润口径中使用 WSC / wholesale cost。
- 适用场景：WF-WSP 广告组、整体广告、分类广告、wf广告、广告利润分析。
- 数据来源：`WF广告数据/广告` 自助表、`wf广告`、`WF广告汇总`、`总广告数据`、公共数据 `王威/新广告` 相关数据集。
- 字段来源：WF-WSP 口径取 `attributed_wholesale_cost_window_view_through_USD_Day_14`；部分新广告表字段为 `WSC`、`WSC_Revenue` 或 `Wholesale_Revenue`，Sponsored Shops / Google Shopping SKU attribution 同时有 retail revenue 字段。FineBI 展示名可能写作 `WSC Revenue` / `Wholesale Revenue`，数据库物理字段多为下划线版。
- 计算公式：WF 广告利润口径为 `SUM_AGG(attributed_wholesale_cost_window_view_through_USD_Day_14)`，为 0 时 FineBI 显示空。
- 粒度：明细可到 `store + date_est + campaign_id/campaign_name + sku`；月报常按 `store + 年月 + campaign + sku`。
- 时间口径：WF 广告日报使用 `date_est`；汇总看板使用 `年月`。
- 过滤条件：按看板筛选器，如店铺、年月区间、日期区间、广告组、分类。
- 汇总规则：求和；不得与 retail sales 混用。
- FineBI 对应看板/组件：`WF-WSP整体广告分析`、`WF-WSP分类广告分析`、`WF-WSP广告组广告分析`、`wf广告`、`wf新广告表现`。
- 注意事项：`attributed_retail_sales_window_view_through_USD_Day_14` 和新广告表里的 `Retail Revenue` 是零售额字段，不等于本口径广告销售额；与 `WSC Revenue`、`WSC`、`Wholesale Revenue` 不能混用。Google Shopping Enhanced Attribution 里同时有 `Sales` 和 `WSC`，广告销售额口径需优先确认展示层取 `WSC` 还是另有字段映射。
- 2026-07-07 过滤边界补充：`advertising_product_report_by_day` 中 `isB2b=true` 和非 `ACTIVE/ACTIVE` 状态组合仍有非零 WSC；默认广告销售额查询不机械过滤 `isB2b`、`campaign_status`、`campaign_is_active`、`product_status`，除非用户明确要求或复刻具体 FineBI 组件过滤。
- 2026-07-07 归因成熟度补充：WF-WSP 广告销售额使用 Day_14 归因字段。FineBI 链路存在 `是否满足14天` 字段，公式为 `DATEDIF(date_est,NOW(),"D")>14`；数据库样本验证最新业务日 `2026-07-06` 有花费但 WSC / orders / units 全为 `0`，`2026-07-05` 起已有非零归因。近 14 天可作为实时观察，但默认不作为最终成熟广告销售额 / ROAS 结论，需标注 `归因未成熟` 或按报表要求排除。
- 2026-07-08 新广告字段补充：`google_shopping_sku_level_attribution.Wholesale_Revenue` 在 `2026-02` 至 `2026-05` 样本中与 `google_shopping.WSC` 汇总一致，可作为 WGS 广告销售额候选；Sponsored Display / Sponsored Shops / Sponsored Video 使用 `WSC_Revenue`。所有新广告表仍需避免和 `Retail_Revenue` / `Retail` / `Sales` 混用。
- 状态：已确认
- 最近更新来源：`docs/business/业务流程需求沉淀.md` 的“FineBI 广告利润口径梳理”；`docs/finebi/FineBI广告数据梳理.md` 的“利润相关口径”。

## Enhanced Attribution

- 业务含义：Google Shopping 归因类型字段，区分 Direct attribution 和 Assisted attribution。
- 适用场景：`wf新广告表现` 中 Google Shopping Enhanced Attribution 分析。
- 数据来源：FineBI `公共数据/王威/新广告/zzdnb_google_shopping_enhanced_attribution`；当前数据库物理表为 `google_shopping_enhanced_attribution`。
- 字段来源：FineBI 展示名为 `Enhanced Attribution`，数据库物理字段为 `Attribution_Type`；指标字段为 `Orders`、`Sales`、`WSC`。
- 口径说明：Direct attribution 表示 Google Shopping 是购买前最后触点；Assisted attribution 表示 Google Shopping 作为早期触点影响购买，但未成为最终归因。该语义来自 FineBI 页面文字说明。
- 汇总规则：`Orders`、`Sales`、`WSC` 可按业务粒度求和；如计算 ROAS/ROI 等比率，仍需按分子分母重算。
- 注意事项：`Sales` 与 `WSC` 不可混用；与 `WF广告汇总` 中 Google Shopping 字段级关系待核实。
- 状态：部分学习
- 最近更新来源：2026-06-17 Chrome / FineBI 公共数据 `zzdnb_google_shopping_enhanced_attribution`。

## 自然销售额

- 业务含义：非广告归因销售额，供广告 vs 自然销售对比。
- 适用场景：WF-WSP 整体广告、分类广告、广告组分析中的自然/总销售对比。
- 数据来源：`总广告数据`、`总广告数据_广告组数据`、`分类目广告数据_广告组数据`。
- 字段来源：`自然销售额`、`分类自然销售额`。
- 计算公式：已确认在 `WF广告数据/广告` 中从 `总广告数据_广告组数据` 或 `分类目广告数据_广告组数据` 求和回补；底层生成公式待核实。
- 粒度：整体为 `store + 年月`；分类为 `store + 年月 + 分类`；广告组主题按广告维度回补月度销售。
- 时间口径：`年月`。
- 过滤条件：店铺、年月、分类等看板筛选器。
- 汇总规则：求和。
- FineBI 对应看板/组件：`WF-WSP整体广告分析` 的 `广告vs自然销售额`；`WF-WSP分类广告分析` 的分类广告 vs 自然销售额；`WF-WSP广告组广告分析`。
- 注意事项：底层自然销售额生成逻辑未完整追到 SQL。
- 状态：待核实
- 最近更新来源：`docs/finebi/FineBI广告数据梳理.md` 的“WF广告数据 分析主题 / 自助数据表”。

## 总销售额

- 业务含义：广告和自然等渠道合计销售额，用于广告销售占比、整体经营表现。
- 适用场景：WF-WSP 整体广告、分类广告、广告组月报。
- 数据来源：`总广告数据`、`总广告数据_广告组数据`、`分类目广告数据_广告组数据`、`zzdnb_wf上架sku每日销售额`。
- 字段来源：`总销售额`、`分类总销售额`、`销售额`。
- 计算公式：FineBI 字段已见；2026-06-18 在 `WF广告数据/广告` 编辑页确认 `总销售额1 = if(ISNULL(销售额),"",销售额/IF(store="WF-FOP",0.93,0.94))`。2026-06-19 数据库核实确认 `销售额` 可从 FineBI 同源物理表 `wf上架sku每日销售额.total_revenue` 按 `store + LEFT(日期,7) + wayfair_sku` 求和回补；该物理表由同名存储过程每日写入，来源为 `wf_supplier_parts_performance` 左连 `平台产品详情表` 补 `链接sku/wayfair_sku`，并对 `GBP/CAD` 通过最新 `货币汇率` 折算。2026-06-30 复核确认同名事件每天调用同名过程，最近执行 `2026-06-30 08:27:00`；目标表无主键 / 索引，`total_revenue` 为字符串金额字段，查询汇总需显式数值转换。
- 组件级口径：`WF-WSP广告组广告分析 -> 广告组活动 -> WSP广告组每月汇总` 及最终 WSP 广告组数据表的 `总销售额` 相关字段，按 `WF广告数据/广告.总销售额1` 口径接入，源头为 `zzdnb_wf上架sku每日销售额 / wf上架sku每日销售额`；`wf每月产品和广告数据汇总` / `wf每月产品和广告数据` 分别对应整体 / 分类月度看板，不作为该广告组活动组件的总销售额主源。
- 粒度：整体 `store + 年月`；分类 `store + 年月 + 分类`；SKU 月销售按 `store + sku + 年月`。
- 时间口径：`年月` 或 `date_est` 汇总到月。
- 过滤条件：店铺、年月、分类、SKU。
- 汇总规则：源销售额在 `store + 月份 + wayfair_sku` 粒度求和；接到广告组明细时按 `store + 月份 + sku = wayfair_sku` 左连。若最终明细含 `campaign`、`bid` 或广告日期拆分，`总销售额1` 会重复出现在多行，向上汇总时必须先按唯一 `store + 月份 + campaign + sku` 去重，或复用 FineBI `AVG_AGG(总销售额1)` 口径，不能直接对拆分行求和。
- FineBI 对应看板/组件：`WF-WSP整体广告分析`、`WF-WSP分类广告分析`、`WF-WSP广告组广告分析`。
- 注意事项：`wf上架sku每日销售额` 比 `wf_supplier_parts_performance` 多 `链接sku`、`wayfair_sku`，更适合按广告 SKU 回补；2026-01 至 2026-06 美国店 `WF-FOP/WF-TD` 与 `wf_supplier_parts_performance` 月店铺汇总一致，`WF-TD-CA/WF-TD-UK` 金额不一致的主要原因是该过程把非美元销售额按最新汇率折算后再写入。2026-06-30 查询时事件已执行但目标表 / 源表最新业务日期均为 `2026-06-27`，排查“当天无销售额”时应先看源表业务日期。本表 `supplier_part` 是销售源 Supplier Part Number / 销售 SKU 层，按产品管理分类可为 `Kit`、`Standard`、`Sellable Component` 或少量 `Nonsellable Component`；不能直接等值关联到 `总账单详情_extract.上架sku`，也不能把所有值都当 `Kit` 父件。总销售额主链路已确认，不代表总成本 / 总利润字段来源已确认。
- 状态：已确认物理源和汇率折算逻辑。

## 广告标签修

- 业务含义：广告组分析中用于滞销/清仓/在售/新品/下架拆分的修正标签。
- 适用场景：`WF-WSP广告组广告分析`、`WSP广告组每月汇总`、标签拆分广告组组件。
- 数据来源：`WF广告数据/广告` 自助表；上游 `广告` 源预览已见 `广告细分标签`、`广告标签`、`垂直标签` 等字段。
- 字段来源：`活动的垂直标签`；数据库侧未发现同名物理列。FineBI 上游 `WF店铺绩效2.0/广告` 已确认从 `zzdnb_公司sku状态表` 按 `UPPER(采购SKU)=UPPER(sku)` 补 `广告细分标签`、`垂直标签` 等字段后生成 `广告标签`，再按 `[extend,isB2b]` 组内字符串拼接为 `活动的垂直标签`。`zzdnb_公司sku状态表` 实际物理表为 `公司sku状态表`，FineBI 更新信息显示无需抽取、自动获取最新数据。
- 上游 `广告标签` 公式：`IF(广告细分标签="新品1-3","新品3个月",IF(OR(广告细分标签="新品4",广告细分标签="新品5",广告细分标签="新品6"),"新品6个月",IF(AND(广告细分标签="新品7"),"在售",垂直标签)))`。
- 计算公式：按字符串包含优先级归类：`清仓` -> `滞销` -> `在售` -> `新品6个月` -> `新品3个月` -> `下架` -> 空。
- 汇总规则：维度字段，不做数值汇总。
- 注意事项：如果一个标签同时包含多个关键词，按上述优先级归类，不是多标签展开。当前 FineBI 链路使用 `公司sku状态表` 当前快照，未按广告月份回溯；若需要历史月度标签，应改接 `公司sku状态表_记录留存` 或 `公司sku状态表_temp_2_记录留存` 的 `留存date` 快照。历史候选源 `上架sku利润表.垂直标签/垂直细分标签` 仍可作为数据库侧标签覆盖对照，但不再作为当前 FineBI 页面直接来源；样本 `WF-TD / 2026-05 / OARI1170` 可经 `wfproduct_management` 桥到多上架 SKU / 多采购 SKU，不能展开后直接汇总广告指标。2026-06-22 主 `广告` 表还确认 `zzdnb_采购sku_model映射表` 会按 `UPPER(采购SKU)=UPPER(采购sku)` 补 `model` 最大值，model 展示和标签归类不要混作一个来源。
- 2026-06-23 补充：留存表可用于历史标签候选，但 2026-01/02 无留存快照，2026-03 起才有月末附近快照。用户已确认本次 Wayfair WSP 2026-01 至 2026-05 广告报表暂不使用历史标签快照，沿用 FineBI 当前看板口径，即使用 `公司sku状态表` 当前快照生成 `活动的垂直标签` / `广告标签修`。

## 是否有价格优势

- 业务含义：判断本品最低前台价相对竞品最低前台价是否有价格优势。
- 适用场景：广告组表现拆分、价格优势维度分析。
- 数据来源：`WF广告数据/广告` 自助表，从 `本竞品价格对比` 回补价格字段。
- 字段来源：`本品最低前台价`、`竞品最低前台价`。
- FineBI 页面原式：`IF(ISNULL(竞品最低前台价), null, IF(竞品最低前台价 < 本品最低前台价, "否", "是"))`，即竞品价低于本品价时为“否”，否则为“是”。
- 当前报表严格业务口径：如果要表达“本品价格必须低于竞品价格才有优势”，应使用 `IF(ISNULL(竞品最低前台价), null, IF(本品最低前台价 < 竞品最低前台价, "是", "否"))`；该口径下本品价等于竞品价时为“否”。
- 汇总规则：维度字段，不做数值汇总。

- 过滤条件：排除 `考虑放弃sku清单`；Wayfair 链接 SKU 排除空值、`NULL`、`无`。
- 汇总规则：按定义重算，不平均下级有货率。

## Amazon 销售、退款、客诉与差评口径

### Amazon 订单销售

- 原始订单行：`amz_orders_allinfo`，粒度为 `Order Item ID`；保留订单状态、MSKU、数量、销售收益、退款和订单总额等源字段。
- 标准化销售：`order_details_tb` 的 `出单平台=AMZ` 分支；销量用 `数量`，销售额用 `总销售额`，采购 SKU 用 `sku修`，业务日期用 `订单日期`。
- 当前边界：两表装载桥为库外 / 待核实；最近 90 天排除取消后仍有 579 个原始订单未命中标准化结果。回答销量前需按目标日期现场检查覆盖。
- Listing 日快照 `领星erp亚马逊销售表` 不是订单销售事实，不得用 `昨日销量 / 销量7_14_30` 替代正式订单销量，除非问题明确要求 Listing 展示指标。

### Amazon 退款

- 原始售后：`领星erp亚马逊售后订单`，售后类型区分退款、退货、换货；源表自带 `退款金额 / 退款总金额 / 退款总费用`。
- 采购 SKU 退款候选：`amz_return_sal`。当前过程按订单价格重算 `退款金额 = 售后数量 * order_details_tb.售价`，并非直接使用领星退款字段。
- 组合 SKU：同一售后 `extend` 拆为多个采购 SKU 时，每个 SKU 行复制完整退款金额。按平台 / 订单汇总前必须先按 `extend` 取一次，或取得正式分摊规则；不能直接 `SUM(amz_return_sal.退款金额)`。
- 新鲜度：源退款已到 2026-07-13，目标只到 2026-06-22；正式查询前同时检查两表最大售后日期和未装载键。
- 月度结果：`退货补件费用.平台=AMZ` 当前包含 Amazon + Walmart，不是纯 Amazon；其销售国家 CASE 未识别 AMZ 加拿大店铺，当前不得直接用于 Amazon 分国家销售 / 退款率。

### Amazon 客诉与差评

- AMZ 客诉事实由退货分析、feedback、replacement 三类来源组成；`ks_num` 是来源行聚合数，客诉订单率分子应按 `COUNT(DISTINCT po_number)` 重算。
- 评论 / 差评不进入 `amz_ks_2`。`差评底表_2_amz` 当前差评阈值为 `评级 <= 3`，与通用差评链的“标准化评分 < 5”不同，不能混用。
- `店铺每月差评率_amz` 的实现为评论发生月差评数除以同月订单数，不是同一批订单的 cohort；当前视图又因排序规则冲突不可执行，仅可作为定义线索。
- feedback 和评论视图均存在一对多 join 放大；正式汇总前分别按 `feedback_id + 上架sku`、`ASIN + Review_ID + 上架sku` 检查重复。
- 状态：原始表、过程和视图定义已确认；正式 Amazon 月度退款、国家口径、差评率和 FineBI 选源仍待核实。
